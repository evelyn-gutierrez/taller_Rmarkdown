1 Tabla de contenidos.

Incluir un índice y especificar (opcionalmente) las subopciones de la tabla de contenidos.

  1. Para incluir una tabla de contenidos, utilice la opción toc.
  2. toc_depth para especificar el número de títulos (por defecto es 3)
  3. toc_float para que el TC sea siempre visible en el margen izquierdo
  4. collapsed (TRUE por defecto) para determinar si la tabla es expandida o compacta por defecto (sólo los encabezados visibles)
  5. smooth_scroll (TRUE por defecto) determina si el TDM se desplaza junto con el desplazamiento de la página
---
title: "Ejemplo"
output:
  html_document:
    toc: true
    toc_depth: 2
    toc_float: true
    collapsed: false
    smooth_scroll: false
---


2 Títulos numerados

Numerar los títulos con la opción number_sections.

---
title: "Ejemplo"
output:
  html_document:
    number_sections: true
---
# (H1)  = 1.
## (H2) = 1.1 (Si no hay H1, dará como resultado 0.1)
  • Para no numerar alguno de los títulos: añadir {-} o {.unnumbered} después del título


3 Pestañas

Al añadir el atributo .tabset al título, todos los subtítulos se convierten en pestañas en lugar de subsecciones

Codigo:

Aquí va el contenido de la pestaña 1.

Contenido de la pestaña 2.

  • También es posible especificar 2 atributos adicionales para las pestañas:
  1. .tabset-fade: las pestañas desaparecerán y aparecerán al pasar de una a otra
  2. .tabset-pills apariencia de las pestañas en “pill” (pastillas):
## Resultados semestrales {.tabset .tabset-fade .tabset-pills}

4 Formatear gráficos

Varias opciones posibles para incluir en el YAML:

  • fig_width y fig_height: 7x5 por defecto.

  • fig_caption: título de los gráficos.

  • dev: soporte para formatos gráficos (png por defecto).

---
title: "Ejemplo"
output:
  html_document:
    fig_width: 5
    fig_height: 4
    fig_caption: true
---

Las opciones de formato del gráfico también pueden especificarse en cada bloque:

fig.width= et fig.height= (en pulgadas), out.width= y out.height= (en %), fig.align= (‘left’, ‘center’, or ‘right’), fig.cap="" (legende), dev= (herramienta gráfica, “png” por defecto o jpeg).

Código:

```{r, out.width='25%', fig.align='center', fig.cap='Une légende'}
plot(cars)
```

Resultado:

plot(cars)

Une légende

4.3 Estilos para documentos word.

Para crear un documento Word a partir de R Markdown, se especifica el formato word_document en la cabecera YAML del documento.

---
title: "Habits"
author: John Doe
date: March 22, 2005
output: 
  word_document:
    reference_docx: my-styles.docx
---
  • Puede crear una plantilla (documento de referencia de estilo) para cambiar el diseño del documento de Word.
  • Esta plantilla debe hacerce un documento Rmd con el formato word_document.
  • A continuación, abra el documento en Word y edite los estilos y guárdelo con otro nombre como archivo `.docx’ y añádalo al YAML.

Ver más en :

  • https://vimeo.com/110804387
  • https://bookdown.org/yihui/rmarkdown-cookbook/word-template.html
  • https://rmarkdown.rstudio.com/articles_docx.html


4.4 Estilos predefinidos para los bloques de resultados

  • Personalice los bloques de código con las opciones class.source y class.output: clases CSS predefinidas que se pueden cambiar o no: “bg-primary”, “bg-success”, “bg-info”, “bg-warning”, “bg-danger”

Código:

```{r class.output="bg-info"}
1 + 1
```

Resultado:

1+1
## [1] 2

Código:

```{r class.output="bg-danger"}
1 + 1
```

Resultado:

1+1
## [1] 2


4.5 Defina sus propios estilos en el archivo .rmd

  1. Define el estilo de una clase resultado:
<style type="text/css">
.resultado {
  background-color: yellow;
  border: 3px solid blue;
  font-weight: bold;
}
</style>
  1. Asigna la clase resultado a los bloques con la opción class.source.

Código

```{r class.source="resultado"}
mtcars[1:5, "mpg"]
```

Resultado:

mtcars[1:5, "mpg"]
## [1] 21.0 21.0 22.8 21.4 18.7

6.1 Compactar bloques de código

Para limitar el espacio que ocupan los bloques de código, es posible “cerrarlos” por defecto con la opción code_fold en el campo html_document de la cabecera o limitar la longitud con un scroll in css

---
title: "Ejemplo"
output:
  html_document:
    code_folding: hide
---

No es posible plegar sólo la salida de código con las opciones del bloque de código, pero se puede hacer con javascript. Sin embargo, es posible limitar la altura del bloque añadiendo un desplazamiento con las opciones attr.source y attr.output.

Código

```{r, attr.output='style="max-height: 100px;"'}
1:300
```

6.2 Chunks en el apéndice

  • Por defecto, los bloques de código y sus resultados aparecen en su lugar en el texto, pero todos los bloques de código pueden añadirse al final del documento en apéndice añadiendo un bloque de código con los argumentos ref.label y la función knitr::all_labels.
  • El argumento echo = FALSE debe añadirse a los bloques de código.

Código

```{r ref.label=knitr::all_labels(), echo=TRUE, eval=FALSE}
```


6.3 Separar bloque de resultados

Servira para presentar el resultado de un bloque de código en otro lugar que no sea directamente bajo el código.

  • Primero, creamos el bloque de código (chunk):
```{r grafico1, dev='png', fig.show='hide'}
plot(cars)
```
  • En otra sección del documento, se llama al resultado de un bloque anterior con la función knitr::fig_chunk y su etiqueta o label: (personalizar_files/figure-html/grafico1-1.png)
```{r}
![Gráfico proveniente de un bloque precedente](personalizar_files/figure-html/grafico1-1.png)`
```


6.4 Descargar todo el código fuente .Rmd

Para añadir un botón de descarga Code>Download Rmd en la parte superior derecha y permitir la descarga del archivo R Markdown se puede usar la opción code_download en el YAML.

---
title: "Ejemplo"
output:
  html_document:
    code_download: TRUE
---


7 Tablas

  • Por defecto, R Markdown presenta las tablas como en la consola de R, pero hay varias opciones para modificar su diseño.

  • La modificación de las tablas puede ser más o menos compleja y puede variar en función del formato de salida.

  • La forma más eficiente de presentar una tabla es con la función knitr::kable() que tiene varios argumentos para establecer la presentación de la tabla.

  • Ver también paquete kableExtra que puede crear tablas para html, pdf y word.

Código

```{r table-cars, results = "asis"}
knitr::kable(cars[1:5, ], caption = 'Título de esta tabla.')
```

Resultado:

knitr::kable(cars[1:5, ], caption = 'Un titre')
Un titre
speed dist
4 2
4 10
7 4
7 22
8 16


7.0.1 Modificar la presentación.

  • ´La opción df_print en YAML sirve para definir la presentación de tablas para los documentos html_document.
  • Valores posibles: default, kable, tibble, paged
---
title: "Ejemplo paged"
output:
  html_document:
    df_print: paged
---
mtcars

Para la opción paged, se pueden especificar varios argumentos en cada código chunk ( max.print, rows.print, cols.print, pages.print,rownames.print … )

Código

```{r cols.print=3, rows.print=3}
mtcars
```

Resultados:

mtcars

  • Esta es una solución útil si no tiene mucho texto que añadir al análisis.

  • También es posible hacer lo contrario y convertir un documento markdown en un script con knit::purl.

Nota: Para usar python, la librería reticulate es útil. Esta sirve para interactuar entre python y R. Esta librería ayudará a traducir los objetos de R a python y viceversa.

Otras formas de incorporar python a R con reticulate son las siguientes:

  • Importar módulos de Python - La función import() permite importar cualquier módulo de Python y llamar a sus funciones directamente desde R.

  • Ejecutar scripts de Python - Con la función source_python() permite ejecutar un script de Python similar a source() de un script de R.

  • Python REPL - Permite acceder a una consola de python desde R. Los objetos que se creean en R estas disponibles en python y viceversa.


  • Ejemplo de uso de python en Rmarkdown:

Comenzamos con un chunk de R para cargar reticulate.

```{r}
# Desde R carga la librería reticulate:
library(reticulate)
# Si tienen python ya inslado en sus computadoras, aquí pueden poner su directorio. Sino, usará una versión instalada por reticulate. 
use_python("C:/Users/EvelynG/AppData/Local/Programs/Python/Python37/python.exe", required=T) 
```

Luego, podemos usar python.

```{python}
import numpy as np
x = 5
y = 7 
z = np.pi
```

Y obtener los valores en R:

```{r}
# Ver valores de x,y y z creados en python: 
py$x
py$y
py$z

# Hacer calculos con ellos:
variable1 <- (py$x + py$y)
variable2  <- py$z*variable1
```

Los objetos creados en R también pueden ser vistos en python:

```{python}
r.variable1
r.variable2
```

11 Contenido interactivo

Con R Markdown también podemos crear contenido interactivo. Hay dos tipos de documentos interactivos que podemos hacer con R Markdown: .

  • HTML widgets para incorporar visualizaciones a partir de librería que hacen uso de javascript.
  • shiny para crear aplicaciones web. Es una herramienta muy versatil. Aprender shiny es un tema amplio, pero podemos utilizar algunos elementos básicos y utilizarlos en Rmarkdown: Rmarkdown + Shiny

11.1 HTML widgets

  • Se pueden crear diversas visualizaciones con los HTML widgets. Se pueden ver todos los widgets en la gallería de HTML widgets.

  • Note que solo es posible tener la interactividad con documentos HTML. Si se crean archivos de otro tipo como pdf, word, no habrá elementos interactivos.

  • Algunos de los paquetes para comenzar son:

    • plotly: Convierte los graficos con ggplot en interactivos. Además tiene una extensa gama de visualizaciones.
    • DT: Tablas interactivas.
    • leaflet: graficos con mapas.
    • dygraph: para graficar series de tiempo
    • highcharter: multiples graficos interactivos.

Ejemplo de una visualización con el paquete leaflet.

Código:

```{r}
library(leaflet)
m <- leaflet() %>%
  addTiles() %>%  # Add default OpenStreetMap map tiles
  addMarkers(lng=174.768, lat=-36.852, popup="The birthplace of R")
m  # Print the map
```

  • Para más detalles puedes revisar HTML widegts, tutorial de Rmarkdown.

11.2 Shiny apps

Shiny sirve para crear aplicaciones web y es un tema amplio a cubrir pero con mucho potencial. Junto a RMarkdown podemos crear elementos Shiny embebidos en nuestros reportes con Rmarkdown.

  • Rmarkdown + Shiny

Para usar shiny en un documento markdown añadimos runtime: shiny al encabezado YAML:

---
title: "Shiny Document"
output: html_document
runtime: shiny
---
