1 Tabla de contenidos.

Incluir un índice y especificar (opcionalmente) las subopciones de la tabla de contenidos.

  1. Para incluir una tabla de contenidos, utilice la opción toc.
  2. toc_depth para especificar el número de títulos (por defecto es 3)
  3. toc_float para que el TC sea siempre visible en el margen izquierdo
  4. collapsed (TRUE por defecto) para determinar si la tabla es expandida o compacta por defecto (sólo los encabezados visibles)
  5. smooth_scroll (TRUE por defecto) determina si el TDM se desplaza junto con el desplazamiento de la página
---
title: "Exemple"
output:
  html_document:
    toc: true
    toc_depth: 2
    toc_float: true
    collapsed: false
    smooth_scroll: false
---


2 Títulos numerados

Numerar los títulos con la opción number_sections.

---
title: "Exemple"
output:
  html_document:
    number_sections: true
---
# (H1)  = 1.
## (H2) = 1.1 (Si no hay H1, dará como resultado 0.1)
  • Para no numerar algunos títulos: añada {-} o {.unnumbered} después del título

4 Formatear gráficos

Varias opciones posibles para incluir en el YAML:

  • fig_width y fig_height: 7x5 por defecto.

  • fig_caption: título de los gráficos.

  • dev: soporte para formatos gráficos (png por defecto).

---
title: "Ejemplo"
output:
  html_document:
    fig_width: 5
    fig_height: 4
    fig_caption: true
---

Las opciones de formato del gráfico también pueden especificarse en cada bloque:

fig.width= et fig.height= (en pulgadas), out.width= y out.height= (en %), fig.align= (‘left’, ‘center’, or ‘right’), fig.cap="" (legende), dev= (herramienta gráfica, “png” por defecto o jpeg).

Código:

```{r, out.width='25%', fig.align='center', fig.cap='Une légende'}
plot(cars)
```

Resultado:

plot(cars)

Une légende


5 Estilos (Styles/CSS)

5.1 Usar temas predefinidos:

  • theme: default, cerulean, journal, flatly, darkly, readable, spacelab, united, cosmo, lumen, paper, sandstone, simplex, yeti (null si no quieres un tema, por ejemplo si quieres usar tu propio css)

  • highlight : para definir la apariencia de la sintaxis: tango, pygments, kate, monochrome, espresso, zenburn, haddock, textmate (u opción null)

Ver galería

---
title: "Ejemplo"
output:
  html_document:
    theme: united
    highlight: tango
---


5.2 Usar estilo .CSS :

---
title: "Ejemplo"
output:
  html_document:
    theme: null
    highlight: null
    css: styles.css
---


5.3 Estilos predefinidos a los bloques de resultados

  • Personalice los bloques de código con las opciones class.source y class.output: clases CSS predefinidas que se pueden cambiar o no: “bg-primary”, “bg-success”, “bg-info”, “bg-warning”, “bg-danger”

Código:

```{r class.output="bg-info"}
1 + 1
```

Resultado:

1+1
## [1] 2

Código:

```{r class.output="bg-danger"}
1 + 1
```

Resultado:

1+1
## [1] 2


5.4 Defina sus propios estilos en el archivo .rmd

  1. Define el estilo de una clase resultado:
<style type="text/css">
.resultado {
  background-color: yellow;
  border: 3px solid blue;
  font-weight: bold;
}
</style>
  1. Asigna la clase resultado a los bloques con la opción class.source.

Código

```{r class.source="resultado"}
mtcars[1:5, "mpg"]
```

Resultado:

mtcars[1:5, "mpg"]
## [1] 21.0 21.0 22.8 21.4 18.7


6 Cerrar bloques de código

Para limitar el espacio que ocupan los bloques de código, es posible “cerrarlos” por defecto con la opción code_fold en el campo html_document de la cabecera o limitar la longitud con un scroll in css

---
title: "Ejemplo"
output:
  html_document:
    code_folding: hide
---

No es posible plegar sólo la salida de código con las opciones del bloque de código, pero se puede hacer con javascript. Sin embargo, es posible limitar la altura del bloque añadiendo un desplazamiento con las opciones attr.source y attr.output.

Código

```{r, attr.output='style="max-height: 100px;"'}
1:300
```

7 Bloques de código en el apéndice

  • Por defecto, los bloques de código y sus resultados aparecen en su lugar en el texto, pero todos los bloques de código pueden añadirse al final del documento en apéndice añadiendo un bloque de código con los argumentos ref.label y la función knitr::all_labels.
  • El argumento echo = FALSE debe añadirse a los bloques de código.

Código

```{r ref.label=knitr::all_labels(), echo=TRUE, eval=FALSE}
```

10 Tablas

  • Por defecto, R Markdown presenta las tablas como en la consola de R, pero hay varias opciones para modificar su diseño.

  • La modificación de las tablas puede ser más o menos compleja y puede variar en función del formato de salida.

  • La forma más eficiente de presentar una tabla es con la función knitr::kable() que tiene varios argumentos para establecer la presentación de la tabla.

  • Ver también paquete kableExtra que puede crear tablas para html, pdf y word.

  • Puede incluir el pie de foto con la opción de pie de foto =

Código

```{r table-cars, results = "asis"}
knitr::kable(cars[1:5, ], caption = 'Título de esta tabla.')
```

Resultado:

speed dist

11 Citas y bibliografía

  • Hay varias formas de insertar las citas, pero la más recomendada es cargar las referencias en un archivo BibTeX (.bib).
  • A continuación, incluya en la cabecera el argumento bibliografía en el YAML y establezca su valor con la ruta del archivo BibTeX en su ordenador.
  • Para cambiar el formato de las citas, descargue el estilo (archivo csl) de la lista [Zotero] (https://www.zotero.org/styles) e inclúyalo en el YAML.
---
title: "Exemple"
output:
  html_document:
bibliography: citations.bib
csl: apa-5th-edition.csl
---

  • Par défaut la bibliographie apparaitra à la fin du document, mais on peut forcer le déplacement des références en utilisant un div: <div id="refs"></div>

  • package citr

  • Après dans texte inclure [@Ref-1] ou [@ref-1; @ref-2; @ref-3] pour parenthèses ou @R-base pour référer à entrée dans le fichier. Pandoc va générer automatiquement une référence à la fin du document.

  • Pour inclure un titre dans la bibliographie qui n’a pas été citée dans le document, il faut définir un champ nocite dans l’entête

---
nocite: '@*'
---

---
nocite: |
  @cit1, @cit2
---

12 Convertir un script en una presentación .Rmd

  • Es posible generar un documento de presentación directamente desde un archivo de script R en RStudio con la función knitr::spin() - (Ctrl / Cmd + Shift + K).
  • Esta es una solución útil si no tiene mucho texto que añadir al análisis.
  • Sin embargo, tendrá que aplicar una sintaxis particular a su archivo de script.
  • También es posible hacer lo contrario y convertir un documento markdown en un script con knit::purl.


  • Ver: https://bookdown.org/yihui/rmarkdown-cookbook/spin.html
  • Ver: https://deanattali.com/2015/03/24/knitrs-best-hidden-gem-spin/


13 Documento Word

Para crear un documento Word a partir de R Markdown, se especifica el formato word_document en la cabecera YAML del documento.

---
title: "Habits"
author: John Doe
date: March 22, 2005
output: 
  word_document:
    reference_docx: my-styles.docx
---
  • Puede crear una plantilla (documento de referencia de estilo) para cambiar el diseño del documento de Word.
  • Esta plantilla debe comenzar con un documento Rmd con el formato word_document.
  • A continuación, abra el documento en Word y edite los estilos y guárdelo con otro nombre como archivo `.docx’ y añádalo al YAML.

Ver:

  • https://vimeo.com/110804387
  • https://bookdown.org/yihui/rmarkdown-cookbook/word-template.html
  • https://rmarkdown.rstudio.com/articles_docx.html


14 Bookdown

  • En RStudio: File -> New Project -> New Directory -> Book Project using bookdown

  • Sirve para escribir documentos largos como un libro a partir de varios documentos R Markdown donde normalmente cada archivo es un capítulo.

  • El primer documento suele llamarse index.Rmd y es el único donde se encuentra el YAML.

  • También debe encontrar en la cabecera: site: bookdown::bookdown_site para que rmarkdown sepa que debe usar bookdown para construir todos los archivos y no sólo un archivo rmd.

  • Se pueden utilizar varios formatos de salida:

    • bookdown::gitbook
    • bookdown::pdf_book
    • bookdown::word_document2
    • bookdown::epub_book

Construye archivos llamando a la función bookdown::render_book(‘index.Rmd’)
